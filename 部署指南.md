# 幽明錄 - 完整部署指南

## 概述

本指南將幫助你完成以下部署：
1. 推送代碼到 GitHub 倉庫
2. 設置 Supabase 數據庫
3. 配置環境變數
4. 遷移數據到 Supabase

---

## 第一步：推送代碼到 GitHub

### 1.1 下載並解壓備份文件

你已經有一個 Git 倉庫備份文件：`youming-record-git.tar.gz`

```bash
# 解壓備份文件
tar -xzf youming-record-git.tar.gz

# 進入目錄
cd youming-record
```

### 1.2 推送到你的 GitHub 倉庫

```bash
# 添加你的 GitHub 倉庫作為遠程地址
git remote add origin https://github.com/zozo971209-pixel/youming-record.git

# 推送代碼
git push -u origin main
```

如果提示輸入用戶名和密碼，請使用：
- 用戶名：你的 GitHub 用戶名
- 密碼：你的 GitHub Personal Access Token（不是登入密碼）

> 如果沒有 Personal Access Token，請在 GitHub Settings > Developer settings > Personal access tokens 中創建一個。

## 第二步：設置 Supabase

### 2.1 創建 Supabase 項目

1. 訪問 https://app.supabase.com
2. 點擊 "New Project"
3. 填寫項目信息：
   - Organization：選擇或創建組織
   - Project name：`youming-record`
   - Database Password：設置一個強密碼（請保存好）
   - Region：選擇離你最近的區域（建議 Southeast Asia - Singapore）
4. 點擊 "Create new project"
5. 等待項目創建完成（約 1-2 分鐘）

### 2.2 執行數據庫結構 SQL

1. 在 Supabase 控制台左側菜單，點擊 "SQL Editor"
2. 點擊 "New query"
3. 複製以下 SQL 內容並粘貼：

```sql
-- 啟用 UUID 擴展
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- 主題表
CREATE TABLE IF NOT EXISTS themes (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  subtitle TEXT,
  description TEXT,
  icon TEXT DEFAULT 'Eye',
  color TEXT DEFAULT 'from-blue-500 to-cyan-500',
  order_index INTEGER DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 區塊表
CREATE TABLE IF NOT EXISTS sections (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  theme_id TEXT REFERENCES themes(id) ON DELETE CASCADE,
  type TEXT CHECK (type IN ('intro', 'methods', 'theory', 'evidence', 'cases', 'sources')),
  title TEXT NOT NULL,
  content TEXT,
  order_index INTEGER DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 引用表
CREATE TABLE IF NOT EXISTS citations (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  theme_id TEXT REFERENCES themes(id) ON DELETE CASCADE,
  section_id UUID REFERENCES sections(id) ON DELETE CASCADE,
  number INTEGER NOT NULL,
  title TEXT NOT NULL,
  author TEXT,
  year INTEGER,
  source TEXT,
  page TEXT,
  url TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 統計數據表
CREATE TABLE IF NOT EXISTS theme_stats (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  theme_id TEXT REFERENCES themes(id) ON DELETE CASCADE UNIQUE,
  success_rate INTEGER DEFAULT 0,
  research_count INTEGER DEFAULT 0,
  participant_count INTEGER DEFAULT 0,
  avg_duration TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 投稿表
CREATE TABLE IF NOT EXISTS submissions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  author TEXT NOT NULL,
  email TEXT,
  category TEXT,
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'approved', 'rejected')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 討論表
CREATE TABLE IF NOT EXISTS discussions (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  author TEXT NOT NULL,
  category TEXT,
  replies INTEGER DEFAULT 0,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 回覆表
CREATE TABLE IF NOT EXISTS replies (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  discussion_id UUID REFERENCES discussions(id) ON DELETE CASCADE,
  content TEXT NOT NULL,
  author TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 管理員設置表
CREATE TABLE IF NOT EXISTS admin_settings (
  id INTEGER PRIMARY KEY DEFAULT 1,
  password_hash TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 插入默認管理員密碼
INSERT INTO admin_settings (id, password_hash) VALUES (1, '@Zozo88888888')
ON CONFLICT (id) DO NOTHING;

-- 創建索引
CREATE INDEX IF NOT EXISTS idx_sections_theme_id ON sections(theme_id);
CREATE INDEX IF NOT EXISTS idx_citations_theme_id ON citations(theme_id);
CREATE INDEX IF NOT EXISTS idx_citations_section_id ON citations(section_id);
CREATE INDEX IF NOT EXISTS idx_submissions_status ON submissions(status);
CREATE INDEX IF NOT EXISTS idx_replies_discussion_id ON replies(discussion_id);

-- 啟用 Row Level Security (RLS)
ALTER TABLE themes ENABLE ROW LEVEL SECURITY;
ALTER TABLE sections ENABLE ROW LEVEL SECURITY;
ALTER TABLE citations ENABLE ROW LEVEL SECURITY;
ALTER TABLE theme_stats ENABLE ROW LEVEL SECURITY;
ALTER TABLE submissions ENABLE ROW LEVEL SECURITY;
ALTER TABLE discussions ENABLE ROW LEVEL SECURITY;
ALTER TABLE replies ENABLE ROW LEVEL SECURITY;
ALTER TABLE admin_settings ENABLE ROW LEVEL SECURITY;

-- 創建 RLS 策略（允許匿名讀取）
CREATE POLICY "Allow anonymous read" ON themes FOR SELECT USING (true);
CREATE POLICY "Allow anonymous read" ON sections FOR SELECT USING (true);
CREATE POLICY "Allow anonymous read" ON citations FOR SELECT USING (true);
CREATE POLICY "Allow anonymous read" ON theme_stats FOR SELECT USING (true);
CREATE POLICY "Allow anonymous read" ON discussions FOR SELECT USING (true);
CREATE POLICY "Allow anonymous read" ON replies FOR SELECT USING (true);

-- 創建 RLS 策略（允許匿名插入投稿和討論）
CREATE POLICY "Allow anonymous insert submissions" ON submissions FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow anonymous insert discussions" ON discussions FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow anonymous insert replies" ON replies FOR INSERT WITH CHECK (true);

-- 創建更新時間觸發器函數
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ language 'plpgsql';

-- 為需要自動更新 updated_at 的表創建觸發器
DROP TRIGGER IF EXISTS update_themes_updated_at ON themes;
CREATE TRIGGER update_themes_updated_at BEFORE UPDATE ON themes
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

DROP TRIGGER IF EXISTS update_sections_updated_at ON sections;
CREATE TRIGGER update_sections_updated_at BEFORE UPDATE ON sections
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();

DROP TRIGGER IF EXISTS update_theme_stats_updated_at ON theme_stats;
CREATE TRIGGER update_theme_stats_updated_at BEFORE UPDATE ON theme_stats
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

4. 點擊 "Run" 執行 SQL

### 2.3 獲取 API 憑證

1. 在 Supabase 控制台左側菜單，點擊 "Project Settings"
2. 點擊 "API" 標籤
3. 複製以下信息：
   - **Project URL**：例如 `https://xxxxxxxxxxxx.supabase.co`
   - **anon public**：以 `eyJ...` 開頭的長字符串

## 第三步：配置環境變數

### 3.1 本地開發環境

1. 在項目根目錄創建 `.env` 文件：

```bash
cp .env.example .env
```

2. 編輯 `.env` 文件，填入你的 Supabase 憑證：

```env
VITE_SUPABASE_URL=https://你的項目ID.supabase.co
VITE_SUPABASE_ANON_KEY=你的anon_key
```

### 3.2 部署環境（Vercel/Netlify）

如果你使用 Vercel 或 Netlify 部署，需要在平台設置中添加環境變數：

**Vercel：**
1. 進入項目設置
2. 點擊 "Environment Variables"
3. 添加：
   - Name: `VITE_SUPABASE_URL`, Value: 你的 URL
   - Name: `VITE_SUPABASE_ANON_KEY`, Value: 你的 Key

**Netlify：**
1. 進入 Site settings
2. 點擊 "Build & deploy" > "Environment"
3. 添加相同的環境變數

## 第四步：遷移數據

### 4.1 安裝遷移腳本依賴



## 常見問題

### Q1: 環境變數設置後仍然顯示 "本地模式"

**原因：** Vite 需要重新加載環境變數

**解決：**
1. 停止開發服務器
2. 重新運行 `npm run dev`
3. 確保 `.env` 文件在項目根目錄

### Q2: 遷移腳本報錯 "relation does not exist"

**原因：** SQL 尚未正確執行

**解決：**
1. 檢查 Supabase SQL Editor 中是否成功創建表
2. 重新執行 SQL 腳本
3. 確保沒有語法錯誤

### Q3: 後台管理無法登入

**原因：** 管理員密碼未正確設置

**解決：**
1. 在 Supabase Table Editor 中檢查 `admin_settings` 表
2. 確保有一條記錄：`id=1, password_hash=@Zozo88888888`
3. 如果沒有，手動插入或重新運行遷移腳本

### Q4: 投稿或討論無法提交

**原因：** RLS 策略阻止了插入

**解決：**
1. 檢查 SQL 中是否正確創建了插入策略
2. 在 Supabase 控制台中檢查 RLS 設置
